RDP Monster — Reseller API

https://manager.rdp.monster/api.php

Quick start — 5 minutes to first VM

Practical recipe from credentials to a managed VM. Each snippet is copy-paste ready.

1. Get your credentials

From your account area: copy your email (the one you log in with) and your API key. Keep the key secret — rotate it if leaked. They go into every request as the username header and as the HMAC seed.

2. Set up environment + token helper

Every call needs the two headers username + token. The token is an HMAC re-computed each call (current UTC hour is baked in, clock drift > 1 h is rejected). Drop this client once, then forget about it:

export RDP_EMAIL="you@example.com"
export RDP_APIKEY="••••••••••••••••"

# Re-export the token before each call (it embeds the current hour).
rdp_token() {
  local d=$(date -u +"%y-%m-%d %H")
  printf "%s" "$RDP_APIKEY" \
    | openssl dgst -sha256 -hmac "$RDP_EMAIL:$d" -hex \
    | awk '{print $2}' \
    | base64 -w 0
}

# Usage
RDP_TOKEN=$(rdp_token)
import crypto from "node:crypto";

const BASE    = "https://manager.rdp.monster/api.php";
const EMAIL   = process.env.RDP_MONSTER_EMAIL;
const API_KEY = process.env.RDP_MONSTER_APIKEY;

function token() {
  const d = new Date()
    .toISOString().slice(2, 13)
    .replace(/[-T:]/g, " ")
    .replace(" ", "-").replace(" ", "-");
  const hex = crypto.createHmac("sha256", `${EMAIL}:${d}`).update(API_KEY).digest("hex");
  return Buffer.from(hex).toString("base64");
}

export async function call(path, init = {}) {
  const sep = path.includes("?") ? "&" : "?";
  const res = await fetch(`${BASE}${path}${sep}_=${Date.now()}`, {
    ...init,
    headers: {
      username: EMAIL, token: token(), Accept: "application/json",
      "Cache-Control": "no-cache, no-store, must-revalidate", Pragma: "no-cache",
      ...(init.headers ?? {}),
    },
    cache: "no-store",
  });
  return { status: res.status, body: await res.json().catch(() => null) };
}
<?php
class RdpMonster {
    private string $base = "https://manager.rdp.monster/api.php";
    public function __construct(private string $email, private string $apiKey) {}
    private function token(): string {
        $hex = hash_hmac("sha256", $this->apiKey, "{$this->email}:" . gmdate("y-m-d H"));
        return base64_encode($hex);
    }
    public function call(string $method, string $path, array $body = []): array {
        $sep = str_contains($path, "?") ? "&" : "?";
        $url = $this->base . $path . $sep . "_=" . round(microtime(true) * 1000);
        $ch  = curl_init($url);
        curl_setopt_array($ch, [
            CURLOPT_RETURNTRANSFER => true, CURLOPT_TIMEOUT => 90,
            CURLOPT_HTTPHEADER => [
                "username: {$this->email}", "token: " . $this->token(),
                "Accept: application/json", "Cache-Control: no-cache, no-store, must-revalidate",
                "Pragma: no-cache", "Content-Type: application/x-www-form-urlencoded",
            ],
        ]);
        if ($method !== "GET") {
            curl_setopt($ch, CURLOPT_CUSTOMREQUEST, $method);
            if ($body) curl_setopt($ch, CURLOPT_POSTFIELDS, http_build_query($body));
        }
        $resp = curl_exec($ch);
        $status = curl_getinfo($ch, CURLINFO_HTTP_CODE);
        curl_close($ch);
        return ["status" => $status, "body" => json_decode($resp, true)];
    }
}

5. Manage the VM

Once you have a serviceId (from response.service.id), all subsequent operations follow /services/{id}/...:

VerbPathWhat it does
GET/services/{id}Live status + spec.
POST/services/{id}/startPower on (idempotent — already-on error treated as success).
POST/services/{id}/stopPower off (idempotent).
POST/services/{id}/rebootReboot.
POST/services/{id}/changepasswordRotate OS account password.
GET/services/{id}/reinstallList available OS templates.
POST/services/{id}/reinstallWipes the VM and reinstalls from a template. Poll status: Rebuilding → Active.
POST/services/{id}/noVncConsoleIssue a noVNC web console URL.
GET/services/{id}/graphs?timeframe=day|week|monthCPU / RAM / network time series.
POST/services/{id}/renewRenew one billing cycle. Charges your credit pool.
POST/services/{id}/upgradeChange product (~50 s — set client timeout ≥ 90 s).

Configurable options

Three options drive the order: VM Template (OS image), Disk Space (size), and IP AddressesXXXX (extra IPv4). All three are passed as configurations[<key>] form fields. Allowed values depend on the product — fetch GET /products and read each product's configOptions map to discover them.

Additional IPv4 — configurations[IP AddressesXXXX]

Integer between 0 and 25. The base product comes with 1 IPv4; this counter adds more. ⚠️ The exact key name has small variants per product (IP AddressesXXXX with four Xs is most common but a few products use IP AddressesXXX with three). Inspect the product's configOptions for the exact key.

IPv4 pricing: each additional IPv4 is billed at $1.75 / month.

Discovering options dynamically (recommended)

Rather than hard-coding the options above, discover them at runtime — the values change per product. Each product exposes a configOptions map keyed by "<key>|<display label>" and each option has a configurableSubOptions[] array with the same "<value>|<display>" pattern. Split on the first | and trim:

const { body } = await call("/products");
const product = body.data.find(p => p.id === 1);

// Parse {"|
$res = $rdp->call("GET", "/products");
$product = current(array_filter($res["body"]["data"] ?? [], fn($p) => $p["id"] === 1));

$options = [];
foreach (($product["configOptions"] ?? []) as $rawKey => $def) {
    $key = trim(explode("|", $rawKey, 2)[0]);
    $values = [];
    foreach (($def["configurableSubOptions"] ?? []) as $sub) {
        if (!empty($sub["hidden"])) continue;
        $parts = explode("|", $sub["optionname"], 2);
        $values[] = [
            "value" => trim($parts[0]),
            "label" => trim($parts[1] ?? $parts[0]),
        ];
    }
    $options[$key] = [
        "type"   => $def["optiontype"]  ?? null,
        "min"    => $def["qtyminimum"]  ?? 0,
        "max"    => $def["qtymaximum"]  ?? 0,
        "values" => $values,
    ];
}
// $options["VM Template"]["values"] = [["value"=>"Windows-Server-2022", "label"=>"Windows Server 2022"], ...]
status, body = call("GET", "/products")
product = next(p for p in (body or {}).get("data", []) if p["id"] == 1)

options = {}
for raw_key, defn in (product.get("configOptions") or {}).items():
    key = raw_key.split("|", 1)[0].strip()
    values = []
    for sub in (defn.get("configurableSubOptions") or []):
        if sub.get("hidden"): continue
        parts = sub["optionname"].split("|", 1)
        values.append({
            "value": parts[0].strip(),
            "label": (parts[1] if len(parts) > 1 else parts[0]).strip(),
        })
    options[key] = {
        "type":   defn.get("optiontype"),
        "min":    defn.get("qtyminimum", 0),
        "max":    defn.get("qtymaximum", 0),
        "values": values,
    }

Overview

HTTP API to provision and manage your RDP / VPS services. All endpoints are mounted under this base URL:

https://manager.rdp.monster/api.php

Paths in the sections below are appended to it.

Conventions

  • • Responses are JSON.
  • • Success typically contains { "result": "success", ... } plus endpoint-specific fields.
  • • Failures typically contain { "result": "error", "error": "<message>" } (sometimes the typo "resut" is observed — clients should tolerate both).
  • • Long-running operations (e.g. /upgrade) are synchronous and may take up to ~50 seconds. Set client read timeouts accordingly (≥ 90 s recommended).
  • • All money-affecting calls (orders, renew, upgrade) are settled against your credit pool, never the end customer.

Building the token header

const crypto = require("node:crypto");

function buildToken(email, apiKey) {
  const datePart = new Date()
    .toISOString()
    .slice(2, 13)
    .replace(/[-T:]/g, " ")
    .replace(" ", "-")
    .replace(" ", "-");
  const hex = crypto
    .createHmac("sha256", `${email}:${datePart}`)
    .update(apiKey)
    .digest("hex");
  return Buffer.from(hex).toString("base64");
}
function buildToken(string $email, string $apiKey): string {
    // gmdate("y-m-d H") yields "26-05-19 14" — the exact datePart used by the server.
    $datePart = gmdate("y-m-d H");
    $hmacKey  = $email . ":" . $datePart;
    $hex      = hash_hmac("sha256", $apiKey, $hmacKey);
    return base64_encode($hex);
}
import hmac, hashlib, base64
from datetime import datetime, timezone

def build_token(email: str, api_key: str) -> str:
    date_part = datetime.now(timezone.utc).strftime("%y-%m-%d %H")
    hmac_key  = f"{email}:{date_part}".encode()
    digest    = hmac.new(hmac_key, api_key.encode(), hashlib.sha256).hexdigest()
    return base64.b64encode(digest.encode()).decode()

Reference clients

Ready-to-paste implementations covering token signing, cache-busting and content-type plumbing for the three most common stacks.

Node.js client

import crypto from "node:crypto";

const BASE    = "https://manager.rdp.monster/api.php";
const EMAIL   = process.env.RDP_MONSTER_EMAIL;
const API_KEY = process.env.RDP_MONSTER_APIKEY;

function token() {
  const datePart = new Date()
    .toISOString()
    .slice(2, 13)
    .replace(/[-T:]/g, " ")
    .replace(" ", "-")
    .replace(" ", "-");
  const hex = crypto.createHmac("sha256", `${EMAIL}:${datePart}`).update(API_KEY).digest("hex");
  return Buffer.from(hex).toString("base64");
}

async function call(path, init = {}) {
  const sep = path.includes("?") ? "&" : "?";
  const res = await fetch(`${BASE}${path}${sep}_=${Date.now()}`, {
    ...init,
    headers: {
      username: EMAIL,
      token: token(),
      Accept: "application/json",
      "Cache-Control": "no-cache, no-store, must-revalidate",
      Pragma: "no-cache",
      ...(init.headers ?? {}),
    },
    cache: "no-store",
  });
  return { status: res.status, body: await res.json().catch(() => null) };
}

// Examples
await call("/services/123456");
await call("/services/123456/reboot", {
  method: "POST",
  headers: { "content-type": "application/x-www-form-urlencoded" }
});
await call("/services/123456/upgrade", {
  method: "POST",
  headers: { "content-type": "application/x-www-form-urlencoded" },
  body: new URLSearchParams({ newProductId: "1002", newCycle: "monthly" }).toString(),
});

PHP client (cURL)

<?php
class RdpMonsterClient {
    private string $base   = "https://manager.rdp.monster/api.php";
    public function __construct(private string $email, private string $apiKey) {}

    private function token(): string {
        $datePart = gmdate("y-m-d H");
        $hex      = hash_hmac("sha256", $this->apiKey, "{$this->email}:{$datePart}");
        return base64_encode($hex);
    }

    public function call(string $method, string $path, array $body = []): array {
        $sep  = str_contains($path, "?") ? "&" : "?";
        $url  = $this->base . $path . $sep . "_=" . round(microtime(true) * 1000);
        $ch   = curl_init($url);
        curl_setopt_array($ch, [
            CURLOPT_RETURNTRANSFER => true,
            CURLOPT_TIMEOUT        => 90,
            CURLOPT_HTTPHEADER     => [
                "username: {$this->email}",
                "token: " . $this->token(),
                "Accept: application/json",
                "Cache-Control: no-cache, no-store, must-revalidate",
                "Pragma: no-cache",
                "Content-Type: application/x-www-form-urlencoded",
            ],
        ]);
        if ($method !== "GET") {
            curl_setopt($ch, CURLOPT_CUSTOMREQUEST, $method);
            if ($body) curl_setopt($ch, CURLOPT_POSTFIELDS, http_build_query($body));
        }
        $resp   = curl_exec($ch);
        $status = curl_getinfo($ch, CURLINFO_HTTP_CODE);
        curl_close($ch);
        return ["status" => $status, "body" => json_decode($resp, true)];
    }
}

$rdp = new RdpMonsterClient($_ENV["RDP_EMAIL"], $_ENV["RDP_APIKEY"]);
print_r($rdp->call("GET", "/services/123456"));
print_r($rdp->call("POST", "/services/123456/reboot"));
print_r($rdp->call("POST", "/services/123456/upgrade", [
    "newProductId" => 1002,
    "newCycle"     => "monthly",
]));

Python client (requests)

import os, hmac, hashlib, base64, time, requests
from datetime import datetime, timezone

BASE    = "https://manager.rdp.monster/api.php"
EMAIL   = os.environ["RDP_MONSTER_EMAIL"]
API_KEY = os.environ["RDP_MONSTER_APIKEY"]

def token() -> str:
    date_part = datetime.now(timezone.utc).strftime("%y-%m-%d %H")
    digest    = hmac.new(f"{EMAIL}:{date_part}".encode(), API_KEY.encode(), hashlib.sha256).hexdigest()
    return base64.b64encode(digest.encode()).decode()

def call(method: str, path: str, body: dict | None = None):
    sep = "&" if "?" in path else "?"
    url = f"{BASE}{path}{sep}_={int(time.time() * 1000)}"
    r = requests.request(
        method, url,
        headers={
            "username": EMAIL,
            "token":    token(),
            "Accept":   "application/json",
            "Cache-Control": "no-cache, no-store, must-revalidate",
            "Pragma":   "no-cache",
            "Content-Type": "application/x-www-form-urlencoded",
        },
        data=body or {},
        timeout=90,
    )
    return r.status_code, r.json() if r.text else None

print(call("GET",  "/services/123456"))
print(call("POST", "/services/123456/reboot"))
print(call("POST", "/services/123456/upgrade",
           {"newProductId": 1002, "newCycle": "monthly"}))
